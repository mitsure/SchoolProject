# Step1 基本集計

## 1. このStepの目的

元データ全体の基本分布を確認し、後続の傷害カテゴリ別解析、テキストマイニング、ジャッカード解析の基準となる集計表を作成する。

このStepでは、各列について件数・割合・順位・欠損状況を整理する。事故原因や受傷機転の抽出、特徴間・事例間の類似度計算は行わない。

---

## 2. フォルダ構成

```text
Meikai/
├─ DB/
│  └─ shougai(2025.01.31).csv
├─ file/
│  └─ Step1_基本集計/
│     ├─ README.md
│     ├─ Step1-1_基本集計_学校種別集計.py
│     ├─ Step1-2_基本集計_学年集計.py
│     └─ Step1-3_基本集計_性別集計.py
└─ CreateData/
   └─ Step1_基本集計/
      ├─ Step1-1_基本集計_学校種別集計.csv
      ├─ Step1-2_基本集計_学年集計.csv
      └─ Step1-3_基本集計_性別集計.csv
```

`CreateData`が空でも、各Pythonの実行時に必要なフォルダを自動生成する。

- 遊具等
- 災害発生時の状況

---

## 4. このStepで使用する解析手法

使用する手法：

- 単純集計
- 件数集計
- 割合計算
- 順位付け
- 欠損値確認
- 必要に応じたクロス集計

### ジャッカード係数

**使用しない。**

理由：

Step1の目的はデータ分布の把握であり、特徴間または事例間の類似性を評価する段階ではないため。

ジャッカード係数を使用する解析は、後続のJaccard解析Stepで行う。

### Step1-1 基本集計：学校種別集計


## 6. 実行順

プロジェクト直下で以下の順に実行する。

```bash
python file/Step1_基本集計/Step1-1_基本集計_学校種別集計.py
python file/Step1_基本集計/Step1-2_基本集計_学年集計.py
python file/Step1_基本集計/Step1-3_基本集計_性別集計.py
```

各Pythonは単独で実行できる。

---

## 7. 出力ファイル

出力先：

```text
CreateData/Step1_基本集計/
```

出力形式：

- CSV
- UTF-8 with BOM
- Excelで文字化けしにくい形式
- 同名ファイルは上書き

---

## 8. 実行前の確認事項

- `DB/shougai(2025.01.31).csv`が存在すること
- Pythonファイルが`file/Step1_基本集計/`にあること
- pandasがインストールされていること
- 元CSVの列名が変更されていないこと

---

## 9. 実行後の確認事項

- 入力件数が元データ総件数と一致するか
- 出力CSVが`CreateData/Step1_基本集計/`に生成されたか
- 件数合計が入力件数と一致するか
- 割合合計が丸め誤差を除き概ね100％か
- 「欠損」や想定外のカテゴリがないか
- 同じ意味の表記が別カテゴリとして分かれていないか

---

## 10. このStepから分かること

- 学校種別の事例数分布
- 学校種・学年別の事例数分布
- 性別ごとの事例数分布
- 各列の欠損状況
- 後続解析で重点的に確認すべき層

---

## 11. このStepでは分からないこと

- 事故原因
- 受傷機転
- 自由記述内の特徴語
- 特徴同士の関連性
- 事例同士の類似性
- テキスト追加による解析結果の変化

これらは後続Stepで解析する。
